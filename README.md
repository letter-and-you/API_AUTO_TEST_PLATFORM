# API测试与监控平台
一款面向开发与测试团队的一站式API测试与监控平台，旨在提升接口测试效率与系统稳定性，覆盖从用例管理到自动告警的全流程API质量保障能力。

## 项目简介
本平台聚焦开发与测试团队的API测试痛点，提供一站式解决方案，核心支持用例管理、同步/异步执行、测试结果分析及异常自动告警能力，通过标准化、自动化的流程，有效提升接口测试效率，保障线上系统接口稳定性。

> **当前开发状态说明**：平台后端功能已全部开发完成并验证可用，原本规划的前后端分离架构中，前端部分仍在迭代完善中，暂未完成前后端对接，现阶段可通过后端API接口完成所有核心功能的调用与验证。

## 核心功能
- **用例管理**：支持API测试用例的全生命周期管理（CRUD），关联项目维度统一管控；
- **灵活执行**：兼顾同步/异步执行模式，适配单次调试、批量测试等不同测试场景；
- **结果分析**：测试结果持久化存储，预留可视化集成能力，可通过接口获取数据进行分析；
- **监控告警**：支持自定义频率的接口巡检，异常场景自动触发邮件告警，提前发现线上问题。

## 技术栈与核心亮点
### 技术架构
- **后端框架**：Django + DRF（Django REST Framework），快速构建高可用的RESTful API服务；
- **数据存储**：PostgreSQL，稳定存储项目、用例、测试结果等结构化数据；
- **异步任务**：Celery + Redis，实现异步任务队列，支撑批量测试、定时监控等高并发场景；
- **可视化规划**：预留Chart.js集成接口，待前端完成后可直观展示测试数据趋势与分布；
- **测试引擎**：基于requests + pytest思想自研，轻量且适配性强。

### 技术亮点
1. **自研测试执行引擎**：原生支持GET/POST等HTTP方法，支持自定义请求头/请求体、响应断言，执行结果实时返回，满足多样化接口测试需求；
2. **异步任务处理**：基于Celery + Redis实现任务解耦，支持长时间运行的批量测试和定时监控任务，大幅提升系统并发处理能力；
3. **可扩展的报告体系**：后端已完成测试成功率、接口耗时、错误详情等数据的持久化存储，预留可视化对接接口，前端完成后可快速集成Chart.js展示通过率趋势、响应时间分布等关键指标；
4. **全链路监控告警**：支持按5分钟~每天的自定义频率自动巡检接口，当接口连续失败时触发邮件通知，保障线上接口可用性；
5. **工程化设计**：代码结构清晰、模块解耦，单元测试覆盖率超70%，具备高可维护性和独立交付能力；
6. **前后端分离规划**：后端接口遵循RESTful规范设计，预留前端对接标准，待前端开发完成后可快速完成整合。

## 项目结构（核心后端）
```
API_AUTO_TEST_PLATFORM/
├── backend/                # 核心后端代码目录
│   ├── accounts/           # 账号权限管理模块
│   ├── apis/               # 对外API接口定义模块
│   ├── common/             # 公共工具/常量/中间件
│   ├── executors/          # 自研测试执行引擎核心模块
│   ├── monitor/            # 监控与告警核心模块
│   ├── projects/           # 项目管理模块
│   ├── reports/            # 测试报告数据处理模块
│   ├── testcases/          # 测试用例CRUD模块
│   ├── manage.py           # Django项目入口
│   └── .env                # 环境配置文件
├── requirements.txt        # 后端依赖清单
└── venv/                   # 虚拟环境目录
```
